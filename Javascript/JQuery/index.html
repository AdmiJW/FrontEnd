Basics of JQuery: Some of its functions

Prevent bugs using Document ready

To use JQuery, you should always include the whole code inside the lambda function passed in as argument of ready function when you select the document, as follows: $(document).ready(function() { //code goes here... } );

Adding class and removing class from elements

We can add and remove classes easily by first using a selector, and using the functions .addClass(" "); and .removeClass(" "); where the name of classes are passed in as arguments. To put multiple classes, just seperate them by space

Adding css and html properties

We can use function .css(property, value); to add css properties to a selected element

We can use function .prop(attribute, value); to add html attributes to a selected element

Manipulating Inner HTML

We use function .html(newHTML); to replace the entire html contained inside the tag of element selected

We use function .text(text); to replace the text inside the tag of element selected. The difference between using .html(newHTML); is that the former renders the tag you put inside, while the latter consider all as text only

Removing, cloning and moving HTML elements

We use function .remove(); to remove the HTML element entirely from the document

We use function .clone(); to clone the selected HTML element, which can be chained with other functions

We use function .appendTo(selected); to remove selected HTML element and append it at the selected HTML element

Selecting parent, children and specific elements

Use function .parent(); to select the selected element's parent element

Use function .children(); to select the selected element's all children element

Selectors

Some Selectors

We can select a specified element with specified id by nesting them and stick the together. For example, $("h1#heading") only selects h1 elements with the id #heading

We can select a element but specified numbering only.

  • $("p:first") select p element but the first one only
  • $("p:last") select p element but the last one only
  • $("p:even") select p element but the even numbered ones only
  • $("p:odd") select p element but the odd numbered ones only
  • $("p:nth-child( f(n) )") select p element but according to the formula passed in f(n) (Eg: f(n) = 3 will select 3rd element, f(n) = 3n will select every 3 element

We can select based on the type attribute by using $(":type"), for example, $(":button") select all elements with the type attribute as button

We can select based on the attribute itself by using $("[attribute]") where attribute is enclosed in square brackets. For example, $("[href]") select all elements with href attribute set. We can even select attribute with specified value with $("[href="value"]").

Events

Click, double click and hover

We can add a click, double click and hover event listener to an element by using .click(function), .dblclick(function) and .hover(function)

The on function

We can use the on function to specify specific mouse events to act on. The definition of the on function is .on(action, function) where the action can be one of follows:

  • click : basic mouse click, mouseup followed by mousedown
  • mouseenter : when the mouse entered the area of the element
  • mouseleave : when the mouse leave the area of the element
  • mousedown : when the mouse button is pressed down
  • mouseup : when the mouse button is released
  • mousemove : when the mouse moves within the area of element

Additionally, in the function argument, we can passed in an optional argument which is an object that contains information of the mouse click

Change event and Submit event

When a selection list or choice has changed a value, we can capture the event and execute something with .change(function). When a form is submitted, then we can use .submit(function) to do something as well. However, you might want to pass in an optional argument and use e.preventDefault(); because the form would always be submitting and your code wouldn't be executed otherwise

DOM Manipulations

Adding, removing and toggling classes and css properties

Adding classes is easily done with .addClass(classes) where classes can be a single class name or multiple of them seperated by spaces, just like what you would do in HTML

Removing classes is done with .removeClass(classes) where if the argument is left blank, it will entirely remove all the classes of the specified element.

In addition to that, classes can be toggled (Added and removed) by using .toggleClass(classes)

CSS can be added one at the time using .css(property, value) or multiple at a time using Objects, .css({property: value, property: value...})

Append, prepend, appendTo, prependTo, before and after

append and prepend will both put the element in the argument into the selected element. The difference is append will put it at the last while prepend will add it at the head. .append(element), and .prepend(element)

appendTo and prependTo is almost like inverse of append and prepend, where the argument is the place you want to append to, and the selector is the element you want to remove from its original place. .appendTo(target), and .prependTo(target)

Unlike append and prepend which places the item inside of the target, before .before(element) and after .after(element) places the element directly before or after the selected element, not inside it.

Empty, Detach and Remove

You can empty the entire selected element's html code by using .empty()

You can remove the selected element from document's flow by using .detach() or .remove(). Both returns the data of element removed. The difference is that .detact() still keeps the event handlers from the removed element and will be useful if the element is supposed to be reinserted at a later time

Prop() or Attr(). Wrap() and WrapAll()

Both .prop(attribute, value) and .attr(attribute, value) will add attribute to the selected element. The difference is .attr() make changes in the HTML parsed code while .prop() make changes with the DOM tree, which can be thinked as actual property. It is more encouraged to use .prop(). Attr will always return string while prop can return other datatypes like boolean etc. Values returned by attr may not be in sync with the actual property value in DOM (after some changes)

.wrap(tag) will wrap all the elements that match the selector in the tag specified, individually (no need for closing tag). .wrapAll(tag) will wrap all the elements that match the selector all in one tag specified
